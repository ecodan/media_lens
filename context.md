## Purpose

Automated Python pipeline that scrapes CNN/BBC/Fox News, extracts article content, runs LLM analysis (5-question media comparison), and deploys static HTML reports to a web server via SFTP. Single operator; public static reader output.

## Architecture

- **Linear batch pipeline**: `harvest → extract → interpret_weekly → summarize_daily → format → deploy`. Each step is independently re-runnable via `-s` flag.
- **Singleton storage**: All I/O routes through `storage.shared_storage` (local filesystem or GCS, switched via `USE_CLOUD_STORAGE`). Never construct file paths without it.
- **Cursor-based incremental processing**: `format` and `deploy` steps use `format_cursor.txt`/`deploy_cursor.txt` to skip unchanged content. Reset with `reset-cursor --all` or `--force-full-*` flags.
- **Hybrid temporal analysis**: Current week = rolling 7-day (fresh); historical weeks = ISO Mon–Sun (stable). `LLMWebsiteInterpreter._is_current_week()` decides which logic applies.
- **Cloud VM (not Cloud Run)**: Playwright requires a persistent container. GCP VM triggered by Cloud Scheduler → HTTP `POST /run` to Flask API on port 8080.
- **LiteLLM routing**: `Agent` uses `litellm.completion()` with model string prefix (`"anthropic/..."` or `"vertex_ai/..."`). Switch provider via `AI_PROVIDER` env var.

## Modules

- **collection**: Playwright scraping + site-specific CSS-selector cleaning; WebpageScraper also used by extraction layer
- **extraction**: LLM headline extraction, article collection, 5-question interpretation, daily summary
- **presentation**: Jinja2 HTML generation + SFTP deploy, both with cursor-based incremental logic
- **storage**: StorageAdapter singleton + JobDir path management + DirectoryManager
- **runner**: CLI orchestration (`runner.py`) + HTTP API (`cloud_entrypoint.py`)

## Conventions

- All imports at top of file; no in-method imports
- Type hints required on all function signatures and variable declarations
- Logger: `create_logger(LOGGER_NAME)` from `common.py`; format `%(asctime)s [%(levelname)s] <%(filename)s:%(lineno)s> %(message)s`
- Paths: `pathlib.Path` objects only — never string concatenation
- Async/await for all I/O; `asyncio.run()` at CLI entry
- File naming: `{site}-clean-extracted.json`, `{site}-article-N.json`, `{site}-interpreted.json`
- Interpreted files go to `intermediate/{job_timestamp}/`, not the job dir itself
- `create_directory()` is a no-op in GCS; do not call it in hot paths (causes 429s)
