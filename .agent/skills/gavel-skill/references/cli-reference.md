## `gavel init`

Initialize gavel in the current project directory. Writes `.gavel/config.json` with the
chosen eval root. Idempotent — safe to re-run. Use `--force` to overwrite.

All eval-touching commands emit a soft yellow warning to stderr if `.gavel/config.json` is absent.

```
Options:
  --eval-root  TEXT  Eval root directory [default: ./evals]
  --force            Overwrite existing config
  --help             Show this message and exit
```

## `gavel oneshot`

OneShot evaluation workflow commands.

### `gavel oneshot create`
Create a new evaluation scaffold.
```
Options:
  --eval        TEXT  Evaluation name [required]
  --type        TEXT  Evaluation type: local or in-situ [default: local]
  --template    TEXT  Scaffold template: default, classification, regression,
                      conversational [default: default]
                      classification/regression wire deterministic judges.
  --eval-root   TEXT  Root directory containing evaluations
                      [env var: GAVEL_EVAL_ROOT]
  --help              Show this message and exit
```

Templates:
- `default` — standard GEval judge scaffold
- `classification` — wires a `classifier` deterministic judge; `field_mapping.expected_output` defaults to `"expected"`
- `regression` — wires a `regression` deterministic judge; `field_mapping.expected_output` defaults to `"expected"`
