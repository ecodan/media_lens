# Module: Storage

> Generated by Cicadas — 2026-03-24.

## Purpose

The Storage module provides a backend-agnostic file I/O abstraction used by every other module in the pipeline. All reads/writes route through a single singleton (`shared_storage`) so the backing store (local filesystem or Google Cloud Storage) can be switched without changing calling code.

## Key Components

### `StorageAdapter` (`storage_adapter.py`)
Singleton file I/O interface.

**Initialization** (via `get_instance()` or `storage.shared_storage`):
- Reads `USE_CLOUD_STORAGE` env var (default: `false`)
- Local backend: reads `LOCAL_STORAGE_PATH` (default: `./working`)
- Cloud backend: reads `GCP_STORAGE_BUCKET`, authenticates via `GOOGLE_APPLICATION_CREDENTIALS` or workload identity
- Initializes a `DirectoryManager` for path generation

**Core methods**:
```python
write_text(path: str, content: str) -> str
read_text(path: str) -> str
write_json(path: str, data: Any) -> str
read_json(path: str) -> Any
write_binary(path: str, content: bytes) -> str
read_binary(path: str) -> bytes
file_exists(path: str) -> bool
list_files(prefix: str) -> List[str]
list_directories(prefix: str) -> List[str]
delete_file(path: str) -> bool
delete_directory(path: str, recursive: bool) -> bool
get_file_modified_time(path: str) -> Optional[datetime.datetime]
get_files_by_pattern(path: str, pattern: str) -> List[str]
```

**Directory path helpers** (delegates to `DirectoryManager`):
```python
get_job_directory(timestamp: Optional[str]) -> str  # → "jobs/YYYY/MM/DD/HHmmss"
get_intermediate_directory(subdir: str) -> str       # → "intermediate/" or "intermediate/{subdir}"
get_staging_directory(subdir: str) -> str            # → "staging/" or "staging/{subdir}"
get_jobs_in_date_range(start_date: str, end_date: str) -> List[str]
```

### `storage.py` — Global singleton
```python
# Usage pattern (all modules):
from src.media_lens.storage import shared_storage
shared_storage.read_text("jobs/2025/03/24/140530/www.cnn.com.html")
```

### `DirectoryManager` (`directory_manager.py`)
Centralized path generation to ensure consistent directory structure.

- `get_job_dir(timestamp) -> str` — generates `jobs/YYYY/MM/DD/HHmmss`
- `get_intermediate_dir(subdir) -> str`
- `get_staging_dir(subdir) -> str`
- `get_jobs_in_date_range(start_date, end_date, storage) -> List[str]` — filters job dirs by date
- `parse_job_timestamp(job_path: str) -> str` — extracts timestamp from hierarchical path

### `JobDir` (`job_dir.py`)
Encapsulates a single job directory's path with rich comparison and grouping support.

**Supported formats**:
- Hierarchical (current): `jobs/YYYY/MM/DD/HHmmss`
- Legacy: `YYYY-MM-DD_HHMMSS`

**Key class methods**:
- `JobDir.from_path(path: str) -> JobDir`
- `JobDir.list_all(storage) -> List[JobDir]` — chronologically sorted
- `JobDir.find_latest(storage) -> Optional[JobDir]`
- `JobDir.group_by_week(job_dirs) -> dict[str, List[JobDir]]` — keyed by `YYYY-WNN`

**Properties**: `storage_path`, `timestamp_str`, `datetime`, `week_key`, `is_hierarchical`

**Comparison**: Supports `<`, `==`, `hash()` based on timestamp — safe for sorting and set operations.

## Interfaces

**Exports**:
- `storage.shared_storage` — used directly by every other module
- `JobDir` — used by `runner.py`, `html_formatter.py`, `interpreter.py` for job discovery

**Dependencies**: None (leaf module — no imports from other `media_lens` modules)

## Key Decisions

- **Singleton pattern**: `StorageAdapter.get_instance()` ensures one GCS client is initialized regardless of how many modules import `shared_storage`. Multiple GCS clients would exhaust connection limits and complicate credential management.
- **All paths are relative strings**: `StorageAdapter` methods accept relative string paths. The adapter resolves them against `LOCAL_STORAGE_PATH` (local) or the GCS bucket root (cloud). Callers should never construct absolute paths.
- **`create_directory()` is a no-op in GCS**: GCS has no real directories. Calling `create_directory()` in GCS-mode returns immediately. Code that calls it is harmless but was removed from hot paths (e.g., `generate_article_page`) to avoid unnecessary GCS API calls that cause 429 rate limiting.
- **`JobDir` supports legacy format**: Some older job directories used `YYYY-MM-DD_HHMMSS`. `JobDir.from_path()` handles both formats transparently.
