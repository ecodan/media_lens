# Module: Extraction

> Generated by Cicadas — 2026-03-24.

## Purpose

The Extraction module is the "brain" of Media Lens. It transforms cleaned HTML into structured data (headlines, article text) and AI-generated insights (5-question analysis, daily summaries). All LLM calls happen here.

## Key Components

### `Agent` / `LiteLLMAgent` (`extraction/agent.py`)
Abstract LLM provider interface with retry logic.

- **Abstract method**: `_invoke_impl(system_prompt, user_prompt, response_format) -> str`
- **`invoke()` method**: Wraps `_invoke_impl` with tenacity retry (5 attempts, exponential backoff 4–60s). Handles `RateLimitError`, `ServiceUnavailableError`, `InternalServerError`. Cleans JSON responses via `_clean_json_response()`.
- **`LiteLLMAgent`**: Concrete implementation using `litellm.completion()`. Model string format: `"anthropic/claude-sonnet-4-5"` or `"vertex_ai/gemini-2.5-flash"`.
- **`create_agent_from_env() -> Agent`**: Factory function. Reads `AI_PROVIDER` env var (`"claude"` or `"vertex"`, default `"vertex"`). Loads API credentials from env.
- **`ResponseFormat` enum**: `TEXT` or `JSON`

### `LLMHeadlineExtractor` (`extraction/headliner.py`)
Extracts top headlines from cleaned HTML using LLM with Chain-of-Thought prompting.

- **`extract(html: str, site: str) -> Dict`**: Returns `{"stories": [...], "metadata": {...}}`
- Each story: `{"headline": "...", "url": "...", "summary": "..."}`
- Extracts top 5 headlines (configurable)
- Uses reflection tags for Chain-of-Thought reasoning
- Validates extracted structure via `_validate_stories()`. Retries up to max attempts on validation failure.
- **`RetryStats` dataclass**: Tracks `total_attempts`, `successful_attempt`, `validation_errors`

### `ArticleCollector` (`extraction/collector.py`)
Fetches full article text from URLs.

- **`collect_articles(urls: list[str]) -> Dict[str, Dict]`**: Returns `{"url": {"title": "...", "text": "..."}, ...}`
- Fallback chain: `WebpageScraper` → direct URL with trafilatura → Wayback Machine
- Note: `WebpageScraper` is used first because trafilatura alone fails on Cloudflare-protected sites (e.g., CNN)

### `ContextExtractor` (`extraction/extractor.py`)
Orchestrates headline extraction + article collection for a job directory.

- **`run(delay_between_sites_secs: int) -> None`**: Main async method
  1. Reads all `*-clean.html` files in job dir
  2. Runs `LLMHeadlineExtractor.extract()` per site
  3. Runs `ArticleCollector.collect_articles()` per site
  4. Saves `{site}-clean-extracted.json` (headlines + metadata)
  5. Saves `{site}-article-N.json` for each article (0-indexed)
- **`_validate_extractions(dir_name)`**: Raises `ArticleExtractionError` if site has < 5 articles

### `LLMWebsiteInterpreter` (`extraction/interpreter.py`)
Answers 5 analysis questions about media coverage using LLM.

**5 questions answered**:
1. What is the most important news right now?
2. What are the biggest issues in the world right now?
3. Is the U.S. President doing a poor/ok/good/excellent job based on media portrayal?
4. Three adjectives describing the U.S. situation
5. Three adjectives describing the President's performance/character

- **`interpret_files(file_paths: List[str]) -> list[dict]`**: Aggregates article content, invokes LLM, returns `[{"question": "...", "answer": "..."}, ...]`
- **`interpret_weeks(sites, specific_weeks) -> list[dict]`**: Collects articles for specified weeks using hybrid date logic
  - Current week: Rolling 7-day window (always fresh)
  - Historical weeks: ISO Mon–Sun boundaries (stable)
- **`_is_current_week(week_key: str) -> bool`**: Determines which date logic to apply
- Output written to `intermediate/{job_timestamp}/{site}-interpreted.json`
- Weekly output: `intermediate/weekly-{YYYY-WNN}-interpreted.json`

### `DailySummarizer` (`extraction/summarizer.py`)
Generates an unbiased daily prose summary from all sources.

- **`generate_summary(articles: list[Path]) -> str`**: Trims each article to 500 words, removes non-ASCII, invokes LLM with Chain-of-Thought (`<thinking>` tags). Returns summary text (≤500 words).
- **`generate_summary_from_job_dir(job_dir_name: str) -> None`**: Reads all article JSON files, calls `generate_summary()`, saves to `{job_dir}/daily_news.txt`
- REASONING_PROMPT uses `<thinking>` tags for chain-of-thought; content after closing tag is the final summary

### Exception Types (`extraction/exceptions.py`)

```python
ExtractionError(Exception)                         # base
ArticleExtractionError(ExtractionError)            # < 5 articles per site
  fields: site, expected, actual
JSONParsingError(ExtractionError)                  # LLM returned unparseable JSON
  fields: raw_response, parse_error
```

## Interfaces

**Exports** (consumed by `runner.py`):
- `ContextExtractor` — `extract` step
- `LLMWebsiteInterpreter` — `interpret` and `interpret_weekly` steps
- `DailySummarizer` — `summarize_daily` step
- `create_agent_from_env()` — creates the LLM agent passed to extractors/interpreters

**Dependencies**:
- `storage.shared_storage` — all file I/O
- `collection.scraper.WebpageScraper` — `ArticleCollector` imports this directly
- `common.py` — logger, `LOGGER_NAME`, utility functions

## Key Decisions

- **LiteLLM as routing layer**: A single `litellm.completion()` call handles both Anthropic and Vertex AI. This means the agent module doesn't need provider-specific SDKs; it just changes the model string prefix.
- **Chain-of-Thought via `<thinking>` tags**: Both `LLMHeadlineExtractor` and `DailySummarizer` use reflection prompting. The final answer is parsed from after the closing `</thinking>` tag.
- **`{site}-clean-extracted.json` naming**: The `-clean-` infix indicates the file was produced from the cleaned HTML. Do not confuse with a hypothetical `{site}-extracted.json`.
- **Interpreted files go to `intermediate/`**: Not to the job dir. This allows multiple job dirs within the same day to contribute to a single interpretation.
- **Minimum 5 articles validation in `ContextExtractor`**: Raises `ArticleExtractionError` if not met. The pipeline can be configured to continue past this error or halt.
